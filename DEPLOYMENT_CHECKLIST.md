## 🎯 Deployment Steps (In Order)

### Phase 1: GitHub Setup
- [ ] Push code to GitHub (main branch)
- [ ] Verify all files are committed
- [ ] Confirm GitHub Actions workflows appear in Actions tab

### Phase 2: Vercel Frontend Deployment
- [ ] Create Vercel account at https://vercel.com
- [ ] Create new project from GitHub repository
- [ ] Configure environment variable: `VITE_ANTHROPIC_API_KEY`
- [ ] Deploy frontend (should auto-deploy on push)
- [ ] Record Vercel:
  - [ ] Project ID
  - [ ] Org ID  
  - [ ] Production URL

### Phase 3: Railway Backend Deployment
- [ ] Create Railway account at https://railway.app
- [ ] Create new project connected to GitHub repo
- [ ] Set root directory to `backend`
- [ ] Add PostgreSQL database service
- [ ] Configure environment variables:
  - [ ] `DATABASE_URL` (auto-generated from PostgreSQL)
  - [ ] `FLASHALPHA_API_KEY`
  - [ ] `NODE_ENV=production`
  - [ ] `PORT=5000`
- [ ] Deploy backend (auto-deploys on push)
- [ ] Record Railway:
  - [ ] Project ID
  - [ ] Backend URL
  - [ ] Service ID

### Phase 4: GitHub Secrets Configuration
- [ ] Add `VERCEL_TOKEN` secret
- [ ] Add `VERCEL_ORG_ID` secret
- [ ] Add `VERCEL_PROJECT_ID` secret
- [ ] Add `VITE_ANTHROPIC_API_KEY` secret
- [ ] Add `RAILWAY_TOKEN` secret
- [ ] Add `RAILWAY_PROJECT_ID` secret
- [ ] Add `DATABASE_URL` secret
- [ ] Add `FLASHALPHA_API_KEY` secret

### Phase 5: Connect Frontend to Backend
- [ ] Update frontend `.env.production` with backend URL
- [ ] Update API client to use `VITE_API_URL`
- [ ] Commit and push changes
- [ ] Verify Vercel redeploy

### Phase 6: Test CI/CD Pipeline
- [ ] Create test branch
- [ ] Open pull request
- [ ] Verify GitHub Actions runs tests
- [ ] Merge to main when tests pass
- [ ] Verify automatic deployment to Vercel & Railway

### Phase 7: Verify Production
- [ ] Open Vercel frontend URL
- [ ] Test market data fetching
- [ ] Verify backend API responds
- [ ] Check database connectivity
- [ ] Monitor logs for errors
