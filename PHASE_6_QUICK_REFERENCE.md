## Deployment Workflow

### To Deploy Changes to Production

1. **Create feature branch**
   ```bash
   git checkout -b feature/your-feature
   ```

2. **Make changes and test locally**
   ```bash
   npm run dev        # Frontend
   npm run dev -w backend  # Backend
   ```

3. **Commit and push**
   ```bash
   git add .
   git commit -m "feat: description"
   git push origin feature/your-feature
   ```

4. **Create Pull Request on GitHub**
   - GitHub Actions automatically runs tests
   - Review test results
   - Once tests pass, get approval

5. **Merge to main**
   ```bash
   git checkout main
   git merge feature/your-feature
   git push origin main
   ```

6. **Automatic deployment starts**
   - Deploy.yml workflow triggers
   - Frontend deploys to Vercel (2 min)
   - Backend deploys to Railway (5 min)
   - Check GitHub Actions for status

## Troubleshooting Quick Answers

### "App not loading" / "Application failed to respond"
1. Check Vercel dashboard for build status
2. Check if environment variables are set
3. Check if backend is responding (curl health check)
4. Try manual redeploy

### "API calls failing" / "Network errors"
1. Verify `VITE_API_BASE_URL` is correct in frontend env vars
2. Check if backend is running (Railway dashboard)
3. Check backend logs for errors
4. Verify CORS is enabled
5. Check if DATABASE_URL is set in backend

### "Database connection error"
1. Go to Railway PostgreSQL service
2. Check if it's running (green status)
3. Copy DATABASE_URL from PostgreSQL variables
4. Paste it into backend service variables
5. Redeploy backend

### "Tests failing in GitHub Actions"
1. Go to GitHub Actions workflow run
2. Click on the failed job
3. Scroll down to see error details
4. Fix the issue locally and push again
